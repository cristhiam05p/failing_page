# Mini local crawler-test website

Node.js + Express application with 3 linked HTML pages and mechanisms to force HTTP 500 responses (one-off and persistent modes).

## Requirements

- Node.js 18+
- npm

## Installation

```bash
npm install
```

## Run

```bash
npm start
```

Default server:

- http://localhost:3000

## Main routes

- `GET /` → Home page with links to `/page-1`, `/page-2`, and `/page-3`
- `GET /page-1` → Blog content about urban gardening
- `GET /page-2` → Blog content about personal finance budgeting
- `GET /page-3` → Blog content about trail running
- `GET /admin` → Simple panel to enable/disable persistent failures

## Force HTTP 500

### Important behavior

When failure mode is enabled, the route **actually throws a runtime error** on the server.
The HTTP 500 response is generated by Express error middleware after that real failure.

### 1) One-off mode (query param)

Add `?force500=1` to any page:

- `/page-1?force500=1`
- `/page-2?force500=1`
- `/page-3?force500=1`

This affects only that request.

### 2) Persistent mode (API)

#### Enable/disable route failure

`POST /api/fail`

JSON body:

```json
{ "path": "/page-2", "enabled": true }
```

To disable it:

```json
{ "path": "/page-2", "enabled": false }
```

#### Check current state

`GET /api/state`

Example response:

```json
{
  "failState": {
    "/page-1": false,
    "/page-2": true,
    "/page-3": false
  }
}
```

## curl examples

```bash
# Check current state
curl http://localhost:3000/api/state

# Enable persistent failure on /page-3
curl -X POST http://localhost:3000/api/fail \
  -H 'Content-Type: application/json' \
  -d '{"path":"/page-3","enabled":true}'

# Verify it now returns 500 due to a thrown runtime error
curl -i http://localhost:3000/page-3

# Disable persistent failure
curl -X POST http://localhost:3000/api/fail \
  -H 'Content-Type: application/json' \
  -d '{"path":"/page-3","enabled":false}'
```
